# Test Cases

This document inventories the current automated test suite and the important device-only scenarios that are still not automated.

Current suite size in this repo:

| Suite | Location | Test files | Test cases |
| --- | --- | ---: | ---: |
| JVM unit tests | `app/src/test` | 11 | 36 |
| Android instrumentation tests | `app/src/androidTest` | 3 | 5 |
| Total | mixed | 14 | 41 |

Primary command used to run everything locally:

```bash
./gradlew testDebugUnitTest connectedDebugAndroidTest
```

## Important Scenarios Still Not Automated

These paths still need emulator, device, or deeper integration coverage beyond the current suite.

| Scenario | Why it still matters |
| --- | --- |
| Fresh install flow through `MainActivity` with the real Android permission dialog | The current suite covers permission evaluation logic, but not the full permission-gated startup flow end to end. |
| Permission revoked in system settings after the app has already been used | The current suite does not verify the `ON_RESUME` permission refresh path in the activity. |
| First sync against a real contacts provider with more than 200 contacts | `ContactsDataSource` chunks detail lookups in batches of 200, but there is no end-to-end provider test for that boundary. |
| Initial import of real device groups and memberships into an empty Room database | Current coverage exercises timestamp policy and mapping logic separately, but not the full repository import path against Room. |
| Initial import when contacts already belong to multiple device groups with ringtones | The suite does not yet verify repository sync, contact mapping, and ringtone resolution together in one integration test. |
| Contacts provider rows with sparse or missing phone, email, address, photo, or ringtone fields | `ContactsDataSource` parsing behavior for incomplete provider rows is not directly covered. |
| `DeviceGroupSyncManager.start()` and `stop()` observer registration and debounce behavior | Current tests cover `syncNow()` error mapping only, not lifecycle or scheduling behavior. |
| Real provider writes for group mirroring, membership writes, and ringtone application | Gateway helpers are partially covered, but there is no full-device verification of actual contacts-provider mutations. |
